# 一、参数配置
修改 Qlib 的 `qrun` 参数，核心就是**修改一个 YAML 配置文件**。这个文件定义了量化研究的完整流程，你所有的调整和优化，最终都会体现在这个文件里。

### ⚙️ `qrun` 如何工作？

`qrun` 是 Qlib 的命令行工具，它会读取一个 YAML 配置文件，然后自动完成从数据处理、模型训练到回测评估的整个流程。你可以把它理解为“量化研究的自动驾驶仪”，只需提供一份详细的“路线图”（YAML 配置文件），它就能自动完成任务。

### 📝 核心配置文件结构解析

一个典型的 `qrun` YAML 配置文件由几个核心部分构成。下面是一个包含了主要配置项的完整示例，我会在注释中说明各部分的含义和修改方法：

```yaml
# ==================== 全局基础设置 ====================
qlib_init:                      # Qlib 初始化配置
  provider_uri: "~/.qlib/qlib_data/cn_data"  # A股数据存储路径，按需修改
  region: cn                     # 市场区域，cn代表A股

# ==================== 数据和特征工程 ====================
data_handler_config: &data_handler_config  # 定义一个配置锚点（可复用）
  start_time: 2010-01-01         # 数据起始日期
  end_time: 2023-12-31           # 数据结束日期
  fit_start_time: 2010-01-01     # 训练数据起始日期
  fit_end_time: 2017-12-31       # 训练数据结束日期
  instruments: "csi300"          # 股票池，如csi300（沪深300），可改为csi500等

task:
  dataset:                       # 数据集配置
    class: DatasetH
    module_path: qlib.data.dataset
    kwargs:
      handler:                   # 数据处理
        class: Alpha158          # 内置的158个技术因子
        module_path: qlib.contrib.data.handler
        kwargs: *data_handler_config
      segments:                  # 时间段划分
        train: ["2010-01-01", "2017-12-31"]
        valid: ["2018-01-01", "2019-12-31"]
        test:  ["2020-01-01", "2023-12-31"]

# ==================== 模型配置 ====================
  model:                         # 模型配置
    class: LGBModel              # 使用的模型，如LGBModel、GRU等
    module_path: qlib.contrib.model.gbdt
    kwargs:
      loss: mse
      colsample_bytree: 0.8      # LightGBM参数，按需调整
      learning_rate: 0.1         # 学习率
      max_depth: 8               # 树的最大深度
      num_leaves: 210            # 叶子节点数
      num_threads: 20            # 并行线程数

# ==================== 策略与回测配置 ====================
port_analysis_config: &port_analysis_config  # 策略与回测配置锚点
  strategy:                      # 策略配置
    class: TopkDropoutStrategy   # 选股策略，如TopkDropoutStrategy
    module_path: qlib.contrib.strategy.strategy
    kwargs:
      topk: 50                   # 持有股票数量
      n_drop: 5                  # 每期调整个数
      signal: <PRED>             # 模型预测结果占位符
  backtest:                      # 回测配置
    start_time: 2020-01-01       # 回测开始日期
    end_time: 2023-12-31         # 回测结束日期
    account: 100000000           # 初始资金（单位：元）
    benchmark: "SH000300"        # 基准指数，沪深300为SH000300
    exchange_kwargs:             # 交易细节配置
      limit_threshold: 0.095     # 涨跌停限制（9.5%）
      deal_price: "close"        # 交易价格，如close、open
      open_cost: 0.0005          # 开仓手续费率
      close_cost: 0.0015         # 平仓手续费率
      min_cost: 5                # 最低手续费（元）

# ==================== 记录与输出 ====================
record:
  - class: SignalRecord          # 记录预测信号
    module_path: qlib.workflow.record_temp
  - class: PortAnaRecord         # 记录投资组合分析
    module_path: qlib.workflow.record_temp
    kwargs:
      config: *port_analysis_config
```

### 🎯 主要修改场景与参数对照表

修改 `qrun` 参数的实质就是调整 YAML 文件中的对应字段。下表总结了几种常见的修改场景及其关键参数：

| 修改场景 | 配置块 (`Section`) | 关键参数与修改方法 | 示例说明 |
| :--- | :--- | :--- | :--- |
| **修改股票池** | `task.dataset` | `instruments`: 在`data_handler_config`中修改。常用值：`csi300`（沪深300）、`csi500`（中证500）、`csi800`（中证800）、`all`（全A股）。 | `instruments: "csi500"` |
| **调整时间段** | `data_handler_config`<br>`task.dataset.kwargs.segments`<br>`port_analysis_config.backtest` | `start_time/end_time`：整体数据范围<br>`fit_start_time/end_time`：训练期<br>`valid/test`：验证/测试期<br>`start_time/end_time`：回测期 | 训练期 `train: ["2015-01-01", "2019-12-31"]`，<br>回测期 `start_time: 2020-01-01` |
| **切换预测模型** | `task.model` | `class`: 修改模型类名<br>`kwargs`: 调整对应的超参数 | 将 `LGBModel` 换为 `GRU` 或 `GAT` 等模型。 |
| **调整策略参数** | `port_analysis_config.strategy.kwargs` | `topk`: 持仓股票数<br>`n_drop`: 调仓时卖出的数量<br>`risk_degree`: 风险偏好等 | `topk: 30` (持有30只股票) |
| **修改回测成本** | `port_analysis_config.backtest.exchange_kwargs` | `open_cost`: 开仓费率<br>`close_cost`: 平仓费率<br>`min_cost`: 最低佣金 | `open_cost: 0.001` (0.1% 开仓费率) |

### 🚀 运行与调试

1.  **运行工作流**：将编辑好的 YAML 文件（如 `my_workflow.yaml`）保存在工作目录下，在终端中运行 `qrun my_workflow.yaml`，即可一键启动流程。
2.  **调试模式**：如需定位问题，可以进入调试模式：
    ```bash
    python -m pdb qlib/cli/run.py examples/benchmarks/LightGBM/workflow_config_lightgbm_Alpha158.yaml
    ```
3.  **查看结果**：运行记录和回测报告会由 Qlib 的 `Recorder` 模块管理，通常会在 `mlruns/` 目录下生成，可以使用 `qrun` 提供的 web 界面进行查看。

### 💡 进阶技巧：使用YAML锚点

YAML 中的锚点（`&`）和引用（`*`）是管理复杂配置的好帮手，能避免重复输入，提升效率和准确性。

*   **定义锚点**：`data_handler_config: &data_handler_config`。
*   **引用锚点**：`kwargs: *data_handler_config`。
*   **好处**：当需要修改一个配置（如数据路径）时，只需修改锚点定义处，所有引用该锚点的地方会自动同步。

### 💎 总结

简单来说，你只需要记住：**修改 `qrun` 参数，就是修改一个 YAML 文件**。`qrun` 工具会负责解析这个文件，并执行其中的指令。掌握了这个 YAML 文件的结构，你就掌握了控制 `qrun` 行为的方法。

# 二、修改股票池

在 Qlib 中定义自定义股票池，核心思路是在 YAML 配置文件中，把 `instruments` 字段的值，从 `csi300` 这样的指数名称，替换为你自己维护的股票代码列表。

这里有三种常用的方法：

### 📝 方法一：直接列举股票代码 (适用于小规模股票池)

如果你的股票池不大，可以直接在 YAML 文件中使用一个列表来指定股票代码，这是最直接的方式。

**1. 修改 `data_handler_config`**
找到 YAML 文件开头的 `data_handler_config` 锚点定义部分，修改其中的 `instruments` 字段。

**2. YAML 配置示例**
```yaml
data_handler_config: &data_handler_config
  start_time: 2010-01-01
  end_time: 2023-12-31
  fit_start_time: 2010-01-01
  fit_end_time: 2017-12-31
  # 直接在配置文件中列举股票代码
  instruments: ['SH600519', 'SZ000858', 'SH600036', 'SZ000333', 'SH600900']
  # ... 其他配置
```

> **重要格式提醒**：
> *   **代码格式**：A股代码需要带上交易所后缀，例如上交所股票为 `SH600519`，深交所股票为 `SZ000858`。
> *   **YAML语法**：这是一个列表，需要使用 YAML 的列表语法（以 `- ` 开头）。

### 📂 方法二：从外部文件读取 (推荐，适用于大型股票池)

当股票数量超过几十只时，将所有代码写在 YAML 文件里会变得难以维护。更好的做法是将它们保存在一个单独的文本文件中，然后在 YAML 里指定文件路径。

**1. 创建股票列表文件**
在你的工作目录（例如 `~/my_a_quant_strategy`）下，创建一个名为 `my_stock_pool.txt` 的文件，每行写入一个股票代码。

```text
# my_stock_pool.txt 文件内容示例
SH600519
SZ000858
SH600036
SZ000333
SH600900
```

**2. 修改 YAML 配置**
在 YAML 文件中，将 `instruments` 的值设置为一个字典，通过 `"file://"` 协议来引用该文件。

```yaml
data_handler_config: &data_handler_config
  start_time: 2010-01-01
  end_time: 2023-12-31
  fit_start_time: 2010-01-01
  fit_end_time: 2017-12-31
  # 从外部文件读取股票池
  instruments: 'file://my_stock_pool.txt' 
  # ... 其他配置
```

### 🚀 方法三：数据过滤 (实现更灵活的动态筛选)

Qlib 还提供了数据过滤器 (`filter`) 功能，可以实现基于数据内容的动态筛选。例如，你可以在数据处理时，自动过滤掉上市不满60天的新股。

**YAML 配置示例**
```yaml
data_handler_config: &data_handler_config
  # ... 其他配置
  filter: &filter
    filter_type: ExpressionDFilter
    rule_expression: "Ref($volume, -60) / $volume > 0.1" 
    filter_start_time: 2010-01-01
    filter_end_time: 2017-12-31
    keep: True 
  # ... 其他配置
```

### 🧪 如何验证配置是否生效？

在终端中运行以下 Python 代码，可以快速验证你的 `instruments` 配置是否被正确解析：

```python
from qlib.data import D
import qlib
from qlib.config import REG_CN

# 初始化 Qlib，provider_uri 请替换为你的数据实际路径
provider_uri = "~/.qlib/qlib_data/cn_data"
qlib.init(provider_uri=provider_uri, region=REG_CN)

# 假设你的 YAML 配置文件路径是 ./workflow.yaml
# 下面这行代码会解析你的 YAML 配置中的 instruments 设定，并返回一个股票列表
instruments = D.instruments(config='file://workflow.yaml')
print(D.list_instruments(instruments=instruments, start_time='2010-01-01', end_time='2017-12-31', as_list=True))
```

配置完成后，你需要确保 Qlib 中已包含这些自定义股票的历史数据。这需要提前将CSV格式的行情数据转换为 Qlib 的二进制格式，或者使用 `get_data.py` 脚本下载对应市场的数据。

# 三、训练范围配置

选择合适的时间段是量化策略成功的关键，它的核心并非追求单一答案，而在于构建一个能经受不同市场环境考验的**稳健评估体系**。

一个完整的回测体系通常由数据范围、训练/验证/测试集和回测窗口三个层次构成。

### 1. 数据范围的起点：历史起点 (Start Time)
确定数据范围的起点是回测的第一步，策略逻辑和市场规则是主要的决定因素。

*   **以市场规则为依据**：A股市场经历了数次制度变革，策略的起点需要确保数据环境与实盘一致。重点关注以下节点：
    *   **2010年**：股指期货上市，引入做空机制。
    *   **2016年**：熔断机制短暂实施，市场波动特性发生变化。
    *   **2019年**：科创板试点注册制，深刻改变发行与定价机制。
    *   **2020年**：创业板涨跌幅放宽至20%，影响相关股票的波动率和流动性。
*   **以策略逻辑为导向**：
    *   **通用策略**：建议从**2010年或2014年**开始。前者覆盖了股指期货时代，后者则能纳入2015年的牛熊转换等多样行情，用以检验策略的长期适应性。
    *   **特定策略**：若策略依赖特定规则（如20%涨跌幅），起点应在规则生效日（如**2020年**）之后，以保证回测环境贴近现实。

### 2. 时间段的三大核心模块
确定了历史起点后，如何切分时间段是关键。通常，整个时间段会被划分为以下三个功能各异的部分。

| 模块 | 作用 | 对应 Qlib YAML 中的字段 (`segments`) |
| :--- | :--- | :--- |
| **训练集 (Train)** | 用于“学习”市场规律、训练模型的历史数据。 | `train` |
| **验证集 (Valid)** | 用于在训练过程中评估模型、调整超参数，防止过拟合。 | `valid` |
| **测试集 (Test)** | 用于最终评估模型表现，模拟“未知”的未来数据，检验泛化能力。 | `test` |

这三个部分在时间上必须严格**先后连续，绝不能有重叠**，以此避免模型“偷看”未来数据（Look-ahead Bias）。

### 3. 如何确定各时间段的最佳长度
确定各段长度需要遵循几个基本原则，同时结合滚动窗口等方法来增强评估的可靠性。

*   **遵循基本法则**：
    1.  **涵盖完整市场周期**：时间段应尽可能包含至少一轮完整的**牛市、熊市、震荡市**周期，以检验策略在不同市场环境下的稳健性。
    2.  **平衡过拟合与欠拟合**：训练时间并非越长越好。研究表明，模型训练存在一个“饱和”区间，超过后性能可能不升反降。短训练期则容易导致模型“欠拟合”，泛化能力弱，未来失效的可能性大。
    3.  **参考行业经验法则**：一个常见的做法是，使用约**5-7年**（如**2010-2016年**）的数据进行模型训练，再用其后约**3-5年**（如**2017-2019年**）的数据进行验证。

*   **使用滚动窗口 (Walk-Forward Analysis)**：为避免一次性划分的偶然性，更稳健的做法是采用滚动窗口交叉验证。其核心思想是，在历史数据上模拟一个不断向前移动的“时光机”，多次重复“训练-验证”的过程，以获得对策略性能更可靠的评估。
    *   **实现方式**：首先设定一个固定的训练窗口长度（如5年），然后让该窗口在时间轴上逐步向后滑动。在每个位置，都用窗口内的数据训练模型，并预测紧接着的下一段时间（如1年），从而得到多个独立的样本外表现。
    *   **主要方法**：
        *   **扩展窗口**：训练集的起点固定，终点不断向后扩展，使用所有历史数据。
        *   **滑动窗口**：训练集的窗口长度固定，随时间整体向前滑动，丢弃最早的数据。

### 4. 一个供参考的实践框架
结合以上原则，下面提供一个你可以直接套用的实践框架：

1.  **设定总数据范围**：根据策略依赖的规则，设定一个足够长的历史数据范围。例如，一个不依赖特殊规则的通用策略，可以从 **2010年1月1日** 开始，到 **2025年12月31日** 结束。
2.  **划分训练/验证/测试集**：在总数据范围内，按时间顺序切分。
    *   **方案一（短期策略）**：`train`: **2020-01-01 至 2023-12-31** (4年)；`valid`: **2024-01-01 至 2024-12-31** (1年)；`test`: **2025-01-01 至 2025-12-31** (1年)。
    *   **方案二（长期稳健）**：`train`: **2015-01-01 至 2020-12-31** (6年)；`valid`: **2021-01-01 至 2022-12-31** (2年)；`test`: **2023-01-01 至 2025-12-31** (3年)。
3.  **实施滚动窗口验证**：在确定整体时间框架后，进一步设计滚动窗口方案。例如，可以设定一个长度为5年的初始训练窗口，并以1年为步长进行滚动，共进行3次验证。这会生成3组独立的回测结果，综合评估策略的稳定性。
4.  **交叉检验**：在多个不同时间段（如2010-2014、2015-2019、2020-2024）上进行上述流程，并比较结果。若策略在不同“经济气候”下的表现差异巨大，则需警惕其稳定性。

### 💎 总结
量化策略时间段的设定没有绝对标准，关键在于构建一个**包含不同市场周期的稳健评估框架**。建议遵循以下流程：

1.  **起点锚定**：根据市场规则，选定合理的起始年份。
2.  **结构搭建**：将数据划分为训练、验证、测试三段，并确保顺序和长度的合理性。
3.  **方法进阶**：引入滚动窗口和多重检验，以评估策略的真实稳健性。

在你的 `.yaml` 配置文件中，修改时间段的语法如下：
```yaml
task:
    dataset:
        kwargs:
            segments: 
                train: ["2015-01-01", "2020-12-31"]
                valid: ["2021-01-01", "2022-12-31"]
                test:  ["2023-01-01", "2025-12-31"]
```

# 四、实盘前的三道必答题
即使数据同分布，以下三个问题若未妥善解决，实盘收益仍可能与回测结果相去甚远。

1️⃣ 过拟合检验：你的模型是“记住”了历史，还是“理解”了规律？
回测中夏普比率高达2.0的策略，实盘可能连0.5都不到。过拟合是量化实盘的“头号杀手”。

检验方法：

滚动窗口验证（Walk-Forward Analysis）：将数据划分为多个连续的时间段，用前期数据训练、后期数据验证，观察绩效稳定性。若各期绩效方差过大，则模型过拟合风险高。

随机打乱标签检验：将预测目标（如下月收益率）随机打乱后重新训练，若模型仍能“预测”出不错的IC，说明模型在学习噪声。

参数敏感性分析：轻微扰动超参数（如树深度±1、学习率±10%），观察回测收益变化幅度。敏感度过高意味着模型不稳定。

2️⃣ 样本外验证：最近的“沉默样本”才是真正的考验
开发模型时，常用2018-2022年数据训练，2023年数据验证。但金融市场具有非平稳性——2023年的市场逻辑可能与2018年截然不同。

最佳实践：

保留最近6-12个月作为“纯样本外”，在模型开发阶段完全不触碰这段数据。最终模型确定后，仅在这段数据上测试一次，结果作为实盘预期收益的基准。

若样本外绩效较回测期大幅下滑（如IC从0.05降至0.02），需谨慎评估市场环境是否发生了结构性变化（如注册制改革、量化交易占比提升等）。
